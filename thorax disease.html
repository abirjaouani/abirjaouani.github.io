OpenCV (Open Source Computer Vision Library)¶

As we said before that an image is read as an array of numbers so each number in this array represents a single pixel.

Data normalisation¶

In this level of work we will explaine the reason of calculating the mean and the std of all the training images. We know that during the process of training our network, we are going to be multiplying (weights) and adding (biases) to these initial inputs to cause activations, which get back-propagated with the gradients to train the model and We want that each feature to have a similar range so that our gradients don’t go out of control (and we only need one global learning rate multiplier). For that reason we have decided to normalize both our train and test datasets.The formula we used to do that is :

                                                $X = {(x-μ)}/{σ}$

NB: for categorical variables, integer encoding is not enough. In fact, using this encoding and allowing the model to assume a natural ordering between categories may result in poor performance or unexpected results.Therefore, we apply one-hot encoding to the integer representation of the categories .

modeling¶

Now we come to The most important part of the work and of a deep learning problem which is how to choose the appropriate training model ? We have two options:

  • Transfer learning: When we transfer(use) a pre-trained model and weights and fit it to our model
  • Create our own model: Here we create our own model from scratch and train its weights ==> this method gives us better control over our model.

What will we use ?

  • We will use the Keras framework to build our model. Keras is a high-level API written in Python. It runs on top Tensorflow, CNTK, or Theano. It is relatively easy to use and understand and is very fast.

Train the model¶

Now we will train our model using the input data and before doing that we will :

  • split our data into 70% training and 30% validation using 'validation_split'.
  • split the input data into batches of 100 images each throw 'batch_size' which refers to the number of training examples utilised in one iteration.
  • run these batches through the model 10 times using 'epoch' which means when the model runs through the entire data once.
